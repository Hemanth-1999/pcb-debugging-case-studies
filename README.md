# PCB Debugging Case Studies

This repository showcases real-world hardware debugging and troubleshooting experience in PCB-based systems, including medical and embedded electronics.

## Tools Used
- Oscilloscope
- Multimeter
- Logic Analyzer

## Skills Demonstrated
- Circuit-level debugging
- Root cause analysis
- Signal tracing
- EMI/noise issue identification
- Hardware validation

---

## Case Study 1: Analog Signal Noise Issue

### Problem
Output signal from an analog circuit was unstable and noisy.

### Investigation
- Checked power supply stability
- Used oscilloscope to observe signal ripple
- Verified grounding and decoupling capacitors

### Root Cause
Improper grounding and insufficient filtering caused noise in the signal path.

### Solution
- Improved grounding strategy
- Added decoupling capacitors
- Verified stable output signal

---

## Case Study 2: Power Supply Failure

### Problem
The board was not powering ON.

### Investigation
- Checked voltage at input and output
- Measured continuity across components
- Inspected for short circuits

### Root Cause
Faulty regulator IC and shorted capacitor.

### Solution
- Replaced faulty components
- Verified correct output voltage
- Confirmed stable operation
